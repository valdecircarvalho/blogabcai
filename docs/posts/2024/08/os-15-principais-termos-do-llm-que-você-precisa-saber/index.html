Os 15 principais termos do LLM que você precisa saber

Os 15 principais termos do LLM que você precisa saber em 2024

Os Large Language Models (LLMs) estão revolucionando a forma como interagimos com a tecnologia. Mas com toda esta inovação surge um novo vocabulário!

Vamos mergulhar em alguns termos essenciais e explicar o que cada um significa:

  • Transformers: Imagine uma poderosa arquitetura de processamento de linguagem. Isso é um transformador! Ele analisa as relações entre palavras no texto, cruciais para que os LLMs entendam e gerem a linguagem

  • Tokens: pense em uma frase como um trem e nas palavras como vagões individuais. Tokens são as unidades básicas de texto que um LLM processa, como palavras ou subpalavras.

  • Chunking: semelhante ao agrupamento de vagões de trem, o chunking envolve dividir o texto em segmentos menores e gerenciáveis ​​para o LLM analisar

  • Indexação: é como criar um catálogo de biblioteca para os enormes conjuntos de dados nos quais os LLMs treinam. A indexação permite a recuperação eficiente de informações específicas.

  • Incorporação: Imagine capturar a essência de uma palavra em um código numérico. Os embeddings representam palavras de uma forma que permite ao LLM compreender suas relações entre si.

  • Pesquisa vetorial: pense em encontrar um livro em uma biblioteca com base em seu tópico. A pesquisa vetorial ajuda os LLMs a encontrar informações semelhantes em seus vastos conjuntos de dados usando embeddings.

  • Banco de dados de vetores: Este banco de dados especializado armazena os códigos numéricos (incorporação) gerados durante o processo de incorporação, permitindo uma pesquisa vetorial eficiente.

  • Inteligência Geral Artificial (AGI): O objetivo final da pesquisa em IA – criar máquinas que possam pensar e aprender como humanos. Os LLMs são um passo significativo nessa direção.

  • Agente LLM: Imagine um LLM treinado especificamente para uma tarefa específica, como escrever diferentes tipos de conteúdo criativo ou responder suas perguntas em um domínio específico. Este é um agente LLM.

  • MOE (Mixture of Experts): Pense em uma equipe de especialistas colaborando. O MOE permite que um LLM aproveite vários modelos especializados menores para melhorar o desempenho em tarefas específicas.

  • Shot Learning: Refere-se a quanta instrução um LLM precisa para aprender uma nova tarefa.

  • Zero-Shot: Nenhum exemplo fornecido, o LLM depende do conhecimento existente.

  • One-Shot: Apenas um exemplo é dado para orientar o LLM.

  • N-Shot: Vários exemplos são fornecidos para o LLM aprender.

  • Ajuste fino com eficiência de parâmetros: LLMs podem ser enormes! Essa técnica permite treinar um modelo menor em cima de um maior, concentrando-se em uma tarefa específica e mantendo o uso de recursos sob controle.

  • RAG (Geração Aumentada de Recuperação): Imagine um LLM que não apenas gera texto, mas também recupera informações relevantes de fontes externas para melhorar sua resposta. Isso é RAG!

  • Prompt Engineering: A arte de elaborar instruções claras e concisas para o LLM alcançar o resultado desejado. Pense nisso como dar ao LLM uma pergunta específica para responder.

  • LoRA quantizado (adaptadores de baixa classificação): Uma técnica para compactar grandes modelos LLM, tornando-os menores e mais rápidos para execução em dispositivos com recursos limitados.

Esta lista apenas arranha a superfície da linguagem LLM! À medida que o campo continua a evoluir, o mesmo acontece com o vocabulário. Mas com esses termos básicos em seu arsenal, você estará no caminho certo para navegar no emocionante mundo dos grandes modelos de linguagem.
